# TaleMori Kit 01 PDF Production Plan

## 1. Final PDF Structure

**Product:** TaleMori Adventure Kit #1: Mori and the Lost Moonlight

**Format:** Printable-only pilot PDF

**Primary page size:** A4 portrait

**Recommended structure:**

1. Cover
2. Parent quick-start guide
3. Story booklet pages 1-8
4. Activity page: Moonlight Path
5. Activity page: Choose Mori's Clue
6. Activity page: Lantern Glow Drawing
7. Little Light Keeper Badge / certificate
8. Parent feedback note or closing page

The pilot should ship as one complete PDF, not multiple files, unless file size becomes a problem.

## 2. Recommended Total Page Count

Recommended pilot count: **14 pages**

Page count breakdown:

- 1 cover page
- 1 parent quick-start guide
- 8 story pages
- 3 activity pages
- 1 badge / certificate page

Optional page:

- 1 closing / feedback page

If production time is tight, remove the optional closing page and keep the PDF at 14 pages total.

## 3. Page-By-Page Layout Plan

### Page 1: Cover

Purpose:

- establish the premium storybook feel
- show Mori as the flagship character
- make the product title clear

Content:

- TaleMori logo / wordmark
- Mori and the Lost Moonlight
- "A ready-to-use story adventure for ages 4-7"
- Mori holding the Lantern of Little Lights

Layout:

- full-page illustration or large framed illustration
- title in upper or lower third
- keep enough clean space for readable title text

### Page 2: Parent Quick-Start Guide

Purpose:

- help a tired parent start within 1 minute
- clarify materials and session length

Content blocks:

- "Start here"
- setup time
- materials needed
- how to use the kit
- short version vs full version
- gentle closing ritual

Layout:

- no dense paragraphs
- use 3-5 simple blocks
- include small icons for read, color, mission, badge

### Pages 3-10: Story Booklet

Use one story page per PDF page.

Story page pattern:

- one main illustration
- short story text
- optional tiny page number
- no large parent instruction block inside the story area

Parent prompts should live in the quick-start guide or as small footer prompts only if layout remains clean.

### Page 11: Moonlight Path Activity

Purpose:

- retell and sequence the story through a simple path

Layout:

- large path with 5 stops
- each stop has a small icon and colorable light
- child instruction at top
- parent prompt at bottom

### Page 12: Choose Mori's Clue Activity

Purpose:

- give the child agency and conversation

Layout:

- three large clue illustrations
- circle / color instruction
- optional blank clue space for older children
- parent prompt at bottom

### Page 13: Lantern Glow Drawing

Purpose:

- create a personal, gentle reflection without forcing disclosure

Layout:

- large lantern outline
- open glow area for drawing or coloring
- short prompt line
- parent prompt at bottom

### Page 14: Little Light Keeper Badge / Certificate

Purpose:

- give the kit a satisfying finish
- create a keepsake

Layout:

- certificate-style page for no-scissors pilot
- badge emblem centered or upper half
- name and date lines
- closing phrase

### Optional Page 15: Closing / Feedback Note

Purpose:

- ask pilot parents what worked
- invite them to share feedback

Layout:

- short thank-you note
- feedback URL or QR code placeholder
- do not include until feedback mechanism is ready

## 4. Page Size Recommendation

Use **A4 portrait first**.

Reasons:

- common for international home printing
- works well for parent guides and activity pages
- simpler than booklet imposition for the first pilot
- easier to export and test quickly

Optional later formats:

- US Letter version
- A5 booklet version
- print-shop-ready version with bleed

Do not build multiple formats for the first internal draft unless parent testing requires it.

## 5. Print Assumptions

Assume parents may print on:

- ordinary home inkjet printer
- ordinary home laser printer
- A4 copy paper
- color or grayscale

Design assumptions:

- no full-bleed requirement for pilot
- keep margins printer-safe
- avoid heavy full-page dark backgrounds on activity pages
- keep story art rich but not ink-hostile if possible
- keep line art clear in grayscale

Recommended safe margins:

- at least 12 mm on all sides
- keep essential text and activity elements inside 15 mm safe area

## 6. Visual Style Guide For The Kit

Tone:

- premium storybook
- gentle woodland adventure
- warm, calm, and magical
- not worksheet-heavy
- not school-like

Palette:

- moss green for Mori and forest accents
- parchment cream for paper backgrounds
- lantern gold for warmth and calls to action
- moonlight silver for the missing glow
- bark brown for grounding details

Texture:

- light watercolor or soft pencil texture
- subtle paper grain
- avoid noisy backgrounds under text

Typography direction:

- story text: readable serif or warm storybook serif
- parent guide and activity instructions: clean humanist sans or highly readable serif
- avoid overly cute fonts
- keep child-facing text large and calm

Character consistency:

- Mori has rounded moss-green hooded cape
- warm cream tunic
- bark-brown boots
- leaf-shaped clasp
- Lantern of Little Lights always visible or narratively present

## 8. Required Icons / Decorative Elements

Core icons:

- lantern
- little light / glow dot
- moonlight shimmer
- leaf
- path marker
- sleepy seed
- silver feather
- moon-shaped stone
- badge emblem

Decorative elements:

- small leaf corners
- soft star specks
- lantern glow rings
- simple path dots
- thin woodland border

Rules:

- decorative elements must not compete with activity space
- icons should be reusable across story, activities, and website previews
- activity icons should work in grayscale

## 9. Activity Page Layout Requirements

Each activity page should include:

- clear page title
- one child-facing instruction
- one parent prompt
- large usable child workspace
- page number or small kit mark

Activity page principles:

- no dense explanation
- no more than one main task per page
- no tiny writing areas
- no required cutting in the default version
- enough white space for coloring and drawing
- instructions readable when printed at home

Recommended text hierarchy:

- title: large
- child instruction: medium and clear
- parent prompt: smaller but readable
- labels: short and simple

## 10. Parent Guide Layout Requirements

The parent guide should fit on one A4 page for the first pilot.

Sections:

- Start here
- Materials
- How to use this kit
- Short version
- Full version
- Helpful prompts
- Closing ritual

Layout:

- use boxed sections or gentle dividers
- avoid long paragraphs
- use bullets
- include 3-4 small icons
- keep tone calm and direct

The parent guide should answer:

- How long does this take?
- What do I need?
- What do I say?
- What if my child does it differently?
- How do we finish?

## 11. Badge / Certificate Layout Requirements

Default pilot format:

- no-scissors certificate page
- child can color and write name
- parent can display or save it

Required copy:

- Little Light Keeper
- I helped Mori carry the little light home.
- Name line
- Date line
- A little light for every brave beginning.

Visual requirements:

- large central badge emblem
- Mori lantern icon
- enough white space for coloring
- premium border, not school certificate clutter
- works in grayscale

Optional later:

- cut-out badge
- sticker format
- printed card

## 14. Export Requirements

PDF export:

- A4 portrait
- RGB for digital download first
- 150-200 dpi images are acceptable for pilot download size
- use embedded fonts or outlined text if needed
- clickable links only if a feedback link is included
- filename: `talemori-kit-01-mori-and-the-lost-moonlight-pilot.pdf`

Image export:

- story illustrations: PNG or high-quality JPG
- activity line art: PNG or SVG if clean and manageable
- website previews: WebP or optimized JPG

File size target:

- ideal: under 20 MB
- acceptable pilot maximum: under 40 MB

If the PDF is too large:

- compress story images first
- keep activity pages mostly vector or line art
- avoid full-resolution unused source images

## 15. QA Checklist Before Sending To Parents

Content QA:

- story title is consistent
- Mori name is spelled consistently
- core phrase appears exactly: "A little light for every brave beginning."
- no school/curriculum tone
- no overclaiming learning or development outcomes
- no scary darkness or high-stakes danger

Parent usability QA:

- parent can understand setup in under 1 minute
- materials are ordinary home materials
- no-scissors version works fully
- activity can run in 20-30 minutes
- prompts are optional and not pressure-heavy

Print QA:

- print one full copy on A4
- check text readability
- check margins
- check grayscale clarity
- check activity writing/drawing space
- check page order
- check file size

Visual QA:

- Mori looks consistent across pages
- lantern glow is clear
- activity icons match story objects
- badge feels like a keepsake
- pages feel like one kit, not separate worksheets

Final QA:

- export final PDF
- open PDF after export
- test on phone and laptop
- print at least story page, activity page, and badge page
- confirm final filename

## 16. MVP Shortcuts Allowed

Allowed for first pilot:

- A4 portrait only
- one combined PDF
- no cut-out version
- no physical packaging
- no interactive PDF fields
- no separate US Letter version
- parent prompts mostly in guide, not every story page
- 8 story pages instead of a longer booklet
- line-art activity pages using simple icons
- certificate-style badge instead of a die-cut badge
- compressed images if visual quality remains acceptable

Allowed visual shortcuts:

- reuse Mori pose if needed
- use repeated border/decorative system
- use simpler background detail on activity pages
- use one consistent page template for story pages

## 17. What Should Not Be Built Yet

Do not build yet:

- subscription system
- school curriculum edition
- login or dashboard
- payment flow before the product offer is final
- automated fulfillment
- collectible production
- cut-out expansion pack
- multiple language versions
- classroom pack
- mobile app
- interactive web activity
- CMS
- analytics-heavy tracking

Do not overbuild the PDF:

- no complex booklet imposition
- no advanced print-shop bleed workflow for pilot
- no large teacher manual
- no multi-activity workbook
- no complicated material requirements

The first objective is to create a polished, printable pilot that parents can understand, print, use, and give feedback on.
